Installation

Follow the steps below to setup your site template:

  • Unzip the downloaded package and open the /bookshelf-main folder to find all the files.
  • Install NPM packages:
    • Run npm install in terminal to install all the uses packages.
    • In case you don't have Node.js & NPM package manager on your machine, here how to add it:
      • Download Node.js from browser. This will automatically add npm with it.
      • Open Command Prompt and check node and npm versions to make sure they have been added.
After installing packages its time to setup your .env file with firebase. Lets see how to do that.
  • Firebase setup:
    • Go to firebase(Here is the link). And Create a project and find your project api key and necessary ids.
    • Now go to your project file and create a env file like this ðŸ‘‰ .env and set the env file like .env.sample.txt file. We created that as an example for you.
    • Set the firebase keys and ids to your newly created .env file.

      **Replace it with your keys and ids. And you are ready to run the project**

  • Development server:
    • Now run npm start to run the project locally. It will navigate you to http://localhost:3000/.
    • The app will automatically reload if you change any of the source files.

Firebase Configuration

We used firebase as backend and authentication. So to upload data to server and get data from server you need to setup firebase. The basic firebase setup you already did and run the project. Now to add data or manage data you must be the admin of the site. We have the authentication verification options. For that you first have to enable email & password authentication from firebase. Lets see how to do that.

  • Go to firebase and open the project you have created.
  • Find Authentication options and enable email/password sign in method
  • Now Click on Add User and write down you email and set a password
  • Now thats the admin email and with that email you can login into site as admin and can make changes you want
  • Log in route isn't linked in anywhere of the site. So to go that page you have to type the path on the browser.
  • The path of the login page is bookshelf-login. You can change the path name as you want from app.js page. Write this path name in browser.

    You'll see this UI there. Put email and pass on field and press login button. If the pass and email match this will login you on the site and will be redirected to home.

  • After the login you'll see some more route which a normal user won't see. Such as add-book, manage-book

    • Add & manage data

  • For adding book data you have to go to add-book page. There you gonna see input field. Fill them with valid details and add data to firebase.
  • But, before that you have to do little job in firebase.
  • Go through the firebase again and find Firebase database there. create database from there.
  • Find storage and setup it to upload images. And now you're all set with firebase
    • As you are all set with firebase you now can add data also can manage data from manage-book page.
  • In manage book page you can update every single data including image and can delete them also if you want.

Template Customization

And to change styles you need to go to sass folder inside src/assets

Process of customize style with SCSS ðŸ‘‡

We have added SASS .scss files in template. Which has been compiling to css. If you know how to use SASS you can change sass files and compile the css as well.

Open the assets/sass/base/_base.scss, assets/sass/base/_typography.scss and Edit the values according to your needs. If you need more Advanced Setup then you can Edit the Respective Files yourself which have been branched inside the same Folder. It is completely at your discretion only to include the Required .scss Files you need to minimize the amount of CSS & including only the Styles of the Blocks you need. This can be setup in your main.scss File.

All colors of template operating from _variable.scss file. Open sass/abstracts/_variable.scss to change colors as you want.

To customize responsive layout you have to open sass/abstracts/mixins.scss. There you can change media queries value as you want.

And All components based scss are in sass/components. Find the page you want to change css there.

Typography

Font Family

We used two different font-family. One for Heading and title and another one for rest. You can simply use another font-family from Google Font by importing it inside assets/sass/base/_typography.scss. 



  <!-- ========== Fonts ========== -->
  <!--====== Heading font ======-->
  @import url("https://fonts.googleapis.com/css2?family=Philosopher:ital,wght@0,400;0,700;1,400;1,700&display=swap")
  <!--====== Body font ======-->
  @import url("https://fonts.googleapis.com/css2?family=Poppins:ital,wght@0,400;0,700;1,400;1,700&display=swap")
  <!-- ========== Add your fonts here or you can replace these two ========== -->

  <!--====== Font Family ======-->
  $font-heading: "philosopher", sans-serif;
  $font-display: "poppins", sans-serif;

After importing fonts inside _typography.scss of index file you just need to add your font family variable in same file. Set your chosen family to $font-heading or $font-display as their value just like the example given up
